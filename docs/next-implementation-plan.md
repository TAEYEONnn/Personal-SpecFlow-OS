# SpecFlow OS 다음 구현 일정

현재 MVP를 실제 업무에서 안정적으로 쓰고, 이후 변경 영향도와 외부 연동으로 확장하는 순서입니다.

## 1주차 — 운영 연결과 품질 기준

- Supabase Production 프로젝트와 Vercel Preview/Production 환경을 연결합니다.
- 실제 OpenAI 호출로 대표 업무 샘플 10개를 컴파일합니다.
- 원문·추론 혼합, 누락 질문, 잘못된 화면 분해를 평가하는 fixture와 회귀 테스트를 만듭니다.
- 모델명, 프롬프트 버전, 처리 시간, 실패 유형을 운영 화면에서 확인할 수 있게 합니다.

완료 기준: 실제 계정으로 로그인하고 10개 샘플을 반복 컴파일해도 기존 문서가 유실되지 않습니다.

## 2주차 — 변경 영향도와 버전 비교

- 요구사항·화면·상태·문구의 안정적인 엔티티 ID 정책을 추가합니다.
- 새 입력이 기존 요구사항과 충돌하거나 영향을 주는 화면을 계산합니다.
- 이전 revision과 현재 revision을 비교하는 변경 내역 화면을 구현합니다.
- 전체 재컴파일 외에 선택 산출물만 다시 생성하는 기능을 추가합니다.

완료 기준: 요구사항 하나를 변경하면 영향받은 화면, 상태, 문서가 이유와 함께 표시됩니다.

## 3주차 — 입력과 전달 확장

- PDF 텍스트 추출과 페이지 번호 기반 출처를 지원합니다.
- Notion OAuth와 프로젝트 문서 내보내기를 추가합니다.
- 내보내기 템플릿을 화면 정의서, QA 체크리스트, 일일보고로 분리합니다.
- 긴 문서의 분할 처리와 비용 예상치를 표시합니다.

완료 기준: PDF 기획서를 넣고 페이지 근거가 포함된 명세를 Notion으로 전달할 수 있습니다.

## 4주차 이후 — Figma 디자인 시스템 매핑

- Figma 라이브러리의 컴포넌트, Variant, 변수와 설명을 읽습니다.
- 화면 요소와 기존 컴포넌트를 추천 연결합니다.
- 기존 컴포넌트 확장, Variant 추가, 신규 컴포넌트, 화면 전용 패턴을 구분합니다.
- 초기에는 Figma 파일을 자동 수정하지 않고 추천과 주석 생성만 지원합니다.

완료 기준: 새 화면 명세마다 사용할 기존 컴포넌트와 부족한 상태가 근거와 함께 제안됩니다.
